BenimÖğretmenim — Google Play (TWA) hazırlığı
============================================

0) Gradle iskeleti (bu repoda)
   - Klasör: apps/twa-android
   - Android Studio → File → Open → apps/twa-android
   - İlk açılışta Gradle sync; ardından Run ile cihazda/emülatörde dene.
   - applicationId: benimogretmenim.com.tr (assetlinks ile aynı olmalı)
   - Varsayılan site: strings.xml içindeki twa_default_url + AndroidManifest VIEW host
   - compileSdk: 36
   - targetSdk: 35
   - Android Gradle Plugin: 8.13.0
   - Önerilen Gradle: 8.13

0.1) APK üretimi (GitHub Actions)
   - Workflow: .github/workflows/android-apk.yml
   - GitHub → Actions → "Build Android APK" → Run workflow
   - Artifact adı: benimogretmenim-android-apks
   - Debug APK: apps/twa-android/app/build/outputs/apk/debug/app-debug.apk
   - Unsigned release APK: apps/twa-android/app/build/outputs/apk/release/app-release-unsigned.apk
   - Debug APK cihazda test içindir. Play Store yayını için signed release gerekir.

0.1.1) Google Play için signed AAB üretimi (GitHub Actions)
   - Workflow: .github/workflows/android-aab.yml
   - GitHub → Repository → Settings → Secrets and variables → Actions → New repository secret
   - Aşağıdaki secret değerleri eklenmeden workflow bilinçli olarak hata verir:
     ANDROID_UPLOAD_KEYSTORE_BASE64
     ANDROID_KEYSTORE_PASSWORD
     ANDROID_KEY_ALIAS
     ANDROID_KEY_PASSWORD
   - Secret değerleri bu makinede oluşturulan yerel dosyadan alınır:
     apps/twa-android/play-upload-key.secrets.local.txt
   - Workflow çalışınca artifact adı:
     benimogretmenim-release-aab
   - Google Play'e yüklenecek dosya:
     app/build/outputs/bundle/release/app-release.aab
   - Keystore ve secret dosyaları git'e eklenmez; .gitignore tarafından korunur.
   - Play Console açıklama, Data Safety ve ekran görüntüsü hazırlığı:
     apps/twa-android/PLAY_STORE_LISTING.md
   - Play Console politika formları (bireysel hesap, finansal özellikler, sağlık):
     apps/twa-android/PLAY_CONSOLE_COMPLIANCE.md
   - İnceleme notu şablonu (İngilizce):
     apps/twa-android/play-review-notes-en.txt

0.2) Yerel APK üretimi
   Gerekenler:
   - JDK 17
   - Android SDK Platform 36
   - Android SDK Build Tools
   - Gradle 8.13 veya Android Studio Gradle sync

   Komut:
   cd apps/twa-android
   gradle :app:assembleDebug

   Çıktı:
   app/build/outputs/apk/debug/app-debug.apk

   Android Studio Gradle 9.x seçerse ve build hata verirse:
   - Settings → Build, Execution, Deployment → Build Tools → Gradle
   - Gradle JDK: Embedded JDK / JetBrains Runtime
   - Gradle dağıtımı olarak 8.13 kullanın veya GitHub Actions artifact yolunu tercih edin.

   Bu makinede JAVA_HOME / java ve Android SDK yoksa yerel build alınamaz; GitHub Actions artifact yolu kullanılır.

1) Android uygulaması (TWA kabuğu)
   - Öneri: Bubblewrap ile üretim (Google’ın resmi aracı):
     npx @bubblewrap/cli@latest init
     (manifest URL olarak canlı sitenizin /manifest.webmanifest adresini verin.)
   - Alternatif: Bu repodaki apps/twa-android iskeletini özelleştir veya Android Studio örnekleri.

2) Play Console
   - Uygulama kimliği (applicationId / package name) sabitlenir.
   - İmzalama: "App signing by Google Play" açıksa Play’in gösterdiği SHA-256 parmak izini kullanın.
   - Yerel debug için ayrıca upload key / debug SHA eklemeniz gerekebilir.
   - İlk AAB yüklemesinden sonra Play Console → Setup → App integrity ekranındaki
     "App signing key certificate" SHA-256 değerini assetlinks.json içine yazın.
   - Upload key SHA-256 değeri imzalama/anahtar takibi için saklanır; Play dağıtımında
     genellikle assetlinks için Play App Signing SHA-256 gerekir.

3) Digital Asset Links
   - Bu repodaki şablon: apps/twa-android/assetlinks.template.json
   - Doldurulmuş içeriği web kökünde şuraya koyun:
     apps/web/public/.well-known/assetlinks.json
   - Şu an canlıda boş dizi [] bırakıldı (yanlış doğrulama riski yok).
   - Yayın öncesi: şablondaki package_name ve sha256_cert_fingerprints gerçek değerlerle değiştirin.
   - Debug APK testinde tam TWA doğrulaması için debug certificate SHA-256 da eklenebilir.
   - Play Store için Google Play App Signing SHA-256 değeri kullanılmalıdır.

3.1) Debug APK test notu
   - assetlinks.json boş kaldığında uygulama yine paketlenir ve siteyi açar.
   - Digital Asset Links doğrulaması tamamlanana kadar Chrome Custom Tab davranışı görülebilir.
   - Tam ekran TWA deneyimi için canlı web kökünde doğru assetlinks.json şarttır.

4) Doğrulama
   - https://developers.google.com/digital-asset-links/tools/generator
   - veya "Statement List Generator and Tester" ile test.

5) Site kökü
   - TWA hedef URL, manifest start_url ile uyumlu HTTPS kök olmalı.
   - Render’da özel alan adı kullanıyorsanız assetlinks ve manifest URL’lerini güncelleyin.
